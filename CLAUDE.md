# psy-helper

AI-ассистент конкретного психолога. **Не** AI-психолог — цифровой помощник для базы знаний по её методу и сопровождения клиентов между сессиями.

**Каноническая спецификация: `tech_spec.md`** (текущая версия v5.2). Это источник истины для всего: продуктового позиционирования, архитектуры, плана работ. Перед любыми изменениями в коде — сверяйся с ним.

## Стек и текущее состояние

Python 3.11 / FastAPI / aiogram / Postgres 16 + pgvector / Redis / Claude API / WhisperX. Деплой: Coolify на Hetzner.

Реализован первый кусок MVP-0 — пайплайн транскрипции (WhisperX + pyannote). Боевой код, FastAPI/aiogram пока нет.

## Локальная разработка (Docker)

Никаких локальных Python/uv установок — всё в контейнере.

**Первый раз:**
```bash
cp .env.example .env
# заполнить HF_TOKEN (см. .env.example для шагов с лицензией pyannote)
docker compose build app          # ~10-15 мин, скачивает torch и whisperx
```

**Транскрипция лекции:**
```bash
# положить файл в data/lectures/
docker compose run --rm app python scripts/transcribe.py data/lectures/lecture_001.m4a
# вывод: data/transcripts/lecture_001/raw.json + metadata.json
```

Первый запуск также скачает Whisper large-v3 (~3 GB) и pyannote-модели — закешируются в Docker volume `models-cache`, повторно не скачиваются.

**Без диаризации (быстрее, не нужен HF_TOKEN):**
```bash
docker compose run --rm app python scripts/transcribe.py data/lectures/file.m4a --no-diarize
```

**Postgres/Redis для будущих этапов:**
```bash
docker compose up -d postgres redis
```

## GPU-режим (Windows + NVIDIA через WSL2)

Для Windows-машины с NVIDIA GPU есть отдельный `Dockerfile.cuda` и override-файл `docker-compose.cuda.yml`. Запуск:
```bash
docker compose -f docker-compose.yml -f docker-compose.cuda.yml build app
docker compose -f docker-compose.yml -f docker-compose.cuda.yml run -d --name psy-batch app python scripts/batch_transcribe.py
```

Override автоматически выставляет `WHISPER_DEVICE=cuda`, `WHISPER_COMPUTE_TYPE=float16`. CPU-конфиг не трогается — на Mac всё работает как раньше.

Подробная инструкция по WSL2 + NVIDIA + сетевая шара с Mac — в чате с разработчиком.

## Батч-обработка лекций

Скрипт `scripts/batch_transcribe.py` — обрабатывает все аудио в `data/lectures/`, идемпотентно (пропускает уже обработанные). Модели грузятся один раз.

```bash
docker compose run -d --name psy-batch app python scripts/batch_transcribe.py
docker logs -f psy-batch
```

## Жёсткие принципы (из ТЗ)

1. **Не AI-психолог.** Бот не ведёт терапию, не делает диагностику, не работает с острыми клиентами. Психолог — супервизор и ответственное лицо.
2. **MVP-стратегия (раздел 1a ТЗ):** MVP-0 справочник из лекций → MVP-1 case-study на разработчике-клиенте с golden standard онтологии графа → MVP-2 безопасные режимы бота без открытого чата → MVP-3 голос + GraphRAG + открытый чат → MVP-4 масштаб. Между этапами — gates, без gate'а следующий этап не начинается.
3. **Двойная роль (раздел 1b ТЗ):** разработчик одновременно клиент этого психолога. Это структурный риск, отражённый архитектурно: только «закрытые» сессии (через N мес после записи + явное закрытие психологом), право вето, ежемесячная сверка состояния терапии. Не предлагать обходы этих ограничений.
4. **Раздел 14 «Критичные НЕ»** — список архитектурных «нет», его уважать буквально.
5. **Юридические/регуляторные вопросы сейчас отложены** по решению владельца. Не пушить на них без явного запроса. Если предложение упирается в legal — пометить и идти дальше.
